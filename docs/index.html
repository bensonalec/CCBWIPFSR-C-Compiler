User Manual

Introduction

We are tasked with creating a compiler for a subset of the C programming language. With this challenge includes figuring out how to implement the three fundamental parts of any compiler: Frontend, Optimization, and the Backend. Our goal with this project is not only to design and implement an optimizing compiler, but to strengthen our ability to design and create software that achieves its purpose and provides adequate documentation to explain how it operates.

How to use the compiler

The compiler is split up into three modules, the frontend, the optimizer, and the backend.
So far only the frontend is implemented, and can so far only be executed by running the frontend.py located in ./src/frontend.
The frontend supports a variety of commandline options such as:
-a
Displays all intermediate outputs produced by both the lexer and parser.
-b
Allows for rebuilding of BNF grammar and parser before the frontend touches the input.
-h
Displays the help message.
-l
Prints out the tokens produced by the lexer in a list.
-p
Prints a tree structure of the tokens after they have been parsed.
-t
Prints a list like tree structure of the tokens after they have been parsed.
